使用无需后期配置的 .dev 域名进行开发

Posted on 2016-12-25 | In Service |

前些日子为了毕设，在 Laracasts 上浏览教程时发现，他们用做开发的域名，就是简单的 .dev 结尾。虽然平时使用 localhost 也无大碍，可是 dev 看着舒服啊，遂找了个时间研究。

首先 dev 不是正规的域名后缀，因此也不用担心会在 Internet 上重名，但也见到过文章说不建议使用这些非正规域名进行调试，不过具体原因已经记不清了，可能主观上我偏向于 dev 吧。

要做的事一共有俩（本文以 MacOS 为例）：

  1. 让 .dev 的域名解析到本地，也就是 127.0.0.1

  2. 在 Web 服务器中按照域名指向目录，这里用 Nginx 实现

  3. 先来实现第一件事，要让本地解析一个域名，方法无非两种：
    1. 使用本机的 Host ，让其在发送至 DNS 之前就返回解析到 127.1
    2. 修改 DNS，让 .dev 返回到本机，其他记录还是使用上级 DNS
    • 方法一，很简便，但是由于 host 是不能使用通配符（至少我没找到方法），每次新建项目只能修改一次 host 文件，不够自动化
    • 方法二，麻烦些，也有几种方法，一种是在 服务器（路由）上将 *.dev 的记录添加到 Dnsmasq 中，还有一种就是在本地搭建 Dnsmasq 等 DNS 工具，本文主要讲本地搭建 Dnsmasq
    1. 安装 Dnsmasq

[bash]brew install dnsmasq[/bash]

2.  ###### 修改配置

这里我还安装了 unbound 作为 Dnsmasq 解析的上层服务，顺便把 tcp 查询 OpenDNS也搭在本地了，具体还可参考使用 CentOS 实现小型网络路由 – 跨过长城
跳过前面那个多余的动作（忽略自动获取的上游 DNS，改用 unbound），基本上就是使用默认即可，主要是添加以下一行，用于匹配域名

PS：截图中我配置了从 dnsmasq.d 中加载所有文件（conf 中修改最后一行注释添加 conf-dir=/usr/local/etc/dnsmasq.d/,*.conf ），实际直接添加此行至 dnsmasq.conf 亦可）

[bash]address=/.dev/127.0.0.1[/bash]

3.  ###### 启动（注意监听端口小于 1024 需要 sudo 权限，这点在 brew 安装时已经说明）

[bash]sudo brew services start dnsmasq[/bash]

4.  ###### 测试（随机.dev 测试，-p参数跟端口）

[bash]dig test.dev @127.0.0.1 -p53[/bash]

5.  ###### 修改系统 DNS 至本地

######

 

  1. 现在访问 .dev 即可解析出本机地址，接下来配置 Nginx，让其自动匹配 ().dev，访问对应文件夹
    1. 安装 nginx 就不细说了，同样是 brew 就行
      1. 修改配置

brew 安装后的info 指出，nginx 会加载（/usr/local/etc/nginx/servers/）内的所有文件，那么我们就在这里新建配置文件

[bash]vi /usr/local/etc/nginx/servers/http-dev.conf[/bash]

这里贴一段配置片段，按需修改：

[bash]
server {

listen 127.0.0.1:5002; # 监听 ipv4 地址端口
listen [::1]:5002; # 监听 ipv6 地址端口
server_name .dev; # 匹配 .dev 域名

## 设置基本路径
set $base_path &quot;/Library/WebServer/Documents&quot;;

## 从访问的 *.dev 中提取 * 作为目录
set $domain $host;
if ($domain ~ &quot;^(.*)\.dev$&quot;) {
      set $domain $1;
}

## 根据 * 确定目录，如果存在 public 即直接访问 public（Laravel 公共目录）
set $domain_path &quot;${domain}&quot;;
if (-d $base_path/$domain/public) {
      set $domain_path &quot;${domain}/public&quot;;
}
if (-f $base_path/$domain/index.html) {
      set $domain_path $domain;
}

## 设置需要访问的目录
root $base_path/$domain_path;
location / {

      index index.html index.php;

      ## 重写规则
      if (-f $request_filename/index.html) {
            rewrite (.*) $1/index.html break;
      }
      if (-f $request_filename/index.php) {
            rewrite (.*) $1/index.php;
      }
      if (!-f $request_filename) {
            rewrite (.*) /index.php;
      }

}

## 配置的 php-fpm 监听地址，将 *.php 文件转发给 fastcgi 处理
location ~ \.php$ {
      fastcgi_pass 127.0.0.1:9000;
      fastcgi_index index.php;
      fastcgi_param SCRIPT_FILENAME $document_root/$fastcgi_script_name;
      include fastcgi_params;
}

## 拒绝访问所有 &quot;.&quot; 开头的文件
location ~ /\. {
      deny all;
}

}
[/bash]

PS：这里使用了 5002 端口，因为上层通过 HAProxy 进行转发，按实际需求修改 80 等

3.  ###### 验证配置

[bash]nginx -t[/bash]

4.  ###### 启动

[bash]sudo brew services start nginx[/bash]

5.  ###### 一些命令，方便调试定位错误

    *   重启服务：

[bash]sudo brew services restart nginx[/bash]

[bash]sudo nginx -s reload[/bash]

*   查看日志：

[bash]tail -f /usr/local/var/log/nginx/error.log[/bash]

*   查看端口占用：

[bash]sudo lsof -i tcp:5003[/bash]

6.  ###### 完成！实际效果如图

Rewrite 工作正常
