Red Hat Enterprise Linux (RHEL) 7.0 基本配置

Posted on 2015-06-07 | In System |

扩展系统Yum源

  • EPEL(Extra Packages for Enterprise Linux)

官方地址：http://dl.fedoraproject.org/pub/epel/OS版本号/x86_64/

Eg: CentOS7 使用 rpm 安装源：

rpm -Uvh http://dl.fedoraproject.org/pub/epel/7/x86_64/e/epel-release-7-5.noarch.rpm

[bash autolinks=”false”]yum install epel-release[/bash]

最常用的源，不多说了。

国内的话，可以使用镜像来提高速度：

*   网易：http://mirrors.163.com/centos/7/extras/x86_64/Packages/epel-release-7-5.noarch.rpm
*   USTC：http://centos.ustc.edu.cn/centos/7/extras/x86_64/Packages/epel-release-7-5.noarch.rpm

  • REMI

官方地址：http://rpms.famillecollet.com/enterprise/版本号/

Eg: CentOS7 使用 rpm 安装源：

[bash autolinks=”false”]rpm -Uvh http://rpms.famillecollet.com/enterprise/remi-release-7.rpm[/bash]

REMI拥有最新的PHP资源，截至2015-06-07 PHP版本为5.6.9，可直接使用yum安装

[bash autolinks=”false”]rpm -Uvh http://rpms.famillecollet.com/enterprise/7/php56/x86_64/php-5.6.9-1.el7.remi.x86_64.rpm[/bash]

⚠：REMI源安装完毕后需要手动启用：

[bash]vi /etc/yum.repos.d/remi.repo[/bash]

修改所需源 enabled=0 为 enabled=1

更改系统默认防火墙

1、关闭firewalld：

  • 停止firewalld

[bash]systemctl stop firewalld.service[/bash]

  • 禁止firewalld开机启动

[bash]systemctl disable firewalld.service[/bash]

2、安装iptables

  • 安装熟悉的iptables：

[bash]yum install iptables-services[/bash]

  • 安装后使用

[bash]iptables -L[/bash]

查看已启用规则，一般文件内默认有22端口规则，

若没有添加默认规则，则需手动添加iptables规则：

[bash]vi /etc/sysconfig/iptables[/bash]

以下例子开启了SSH，HTTP，MYSQL端口

[code collapse=”true”]*filter

:INPUT ACCEPT [0:0]

:FORWARD ACCEPT [0:0]

:OUTPUT ACCEPT [0:0]

-A INPUT -m state –state ESTABLISHED,RELATED -j ACCEPT

-A INPUT -p icmp -j ACCEPT

-A INPUT -i lo -j ACCEPT

-A INPUT -m state –state NEW -m tcp -p tcp –dport 22 -j ACCEPT

-A INPUT -m state –state NEW -m tcp -p tcp –dport 80 -j ACCEPT

-A INPUT -m state –state NEW -m tcp -p tcp –dport 3306 -j ACCEPT

-A INPUT -j REJECT –reject-with icmp-host-prohibited

-A FORWARD -j REJECT –reject-with icmp-host-prohibited

COMMIT[/code]

编辑完文件后，按ESC，输入 :wq! #保存退出，回车确认

  • 重启防火墙使配置生效

[bash]systemctl restart iptables.service[/bash]

  • 确认设置无误后，设置开机启动，若设置有误则重启后重新编辑配置文件

[bash]systemctl enable iptables.service[/bash]

安装组件

1、安装网页服务器，解释语言

  • Apache Httpd:

[bash]yum install httpd[/bash]

启动httpd:

[bash]systemctl start httpd.service[/bash]

停止httpd:

[bash]systemctl stop httpd.service[/bash]

重启httpd:

[bash]systemctl restart httpd.service[/bash]

设置httpd开机启动:

[bash]systemctl enable httpd.service[/bash]

  • PHP:

[bash]yum install php[/bash]

默认安装版本较旧(5.4.x)，若需最新版本，请安装php后继续添加REMI源安装”PHP+版本”：

Eg: 安装PHP 5.6.9:

[bash]yum install php56[/bash]

Eg: 安装PHP 7.0+:

[bash]yum install php70[/bash]

按需安装PHP组件:

[bash]yum install php-mysql php-gd libjpeg* php-ldap php-odbc php-pear php-xml php-xmlrpc php-mhash[/bash]

按需安装PHP 7组件:

[bash]yum install php70-php-mysqlnd php70-php-gd libjpeg* php70-php-ldap php70-php-pear php70-php-xml php70-php-xmlrpc[/bash]

用于开发的话，推荐安装xDebug:

[bash]yum install php-pecl-xdebug[/bash]

2、安装数据库服务器

  • MariaDB:

[bash]yum install mariadb mariadb-server[/bash]

拷贝配置文件:（注：重名请覆盖）

安装后默认提供5个配置文件，选择合适的复制至/etc/my.cnf

[code]my-huge.cnf

my-innodb-heavy-4G.cnf

my-large.cnf

my-medium.cnf

my-small.cnf[/code]

Eg: 配置中型数据库：

[bash]cp /usr/share/mysql/my-medium.cnf /etc/my.cnf[/bash]

启动MariaDB:

[bash]systemctl start mariadb.service[/bash]

停止MariaDB:

[bash]systemctl stop mariadb.service[/bash]

重启MariaDB:

[bash]systemctl restart mariadb.service[/bash]

设置开机启动:

[bash]systemctl enable mariadb.service[/bash]

安装完成并启动数据库后，执行数据库自带初始化安装：

[bash]mysql_secure_installation[/bash]

一路Y与回车，设置数据库root密码，直到出现 Thanks for using MariaDB!

至此MariaDB安装结束，若需要远程登录（%权限）则：

[bash]mysql -u root -p[/bash]

输入密码后按回车，进入mysql控制台，接着输入

[sql]use mysql;[/sql]

[sql]update user set host = ‘%’ where user = ‘root’;[/sql]

接着会报错找到多个root用户，不用管他，还是会修改第一个root的访问权限为’%’

最后别忘记刷新权限，使修改生效：

[sql]FLUSH PRIVILEGES;[/sql]

配置组件

1、配置Apache Httpd

  • 一般情况下修改Httpd配置文件：

[bash]vi /etc/httpd/conf/httpd.conf[/bash]

选项：

[code]Options Indexes FollowSymLinks[/code]

修改为（禁止列目录）：

[code]Options FollowSymLinks[/code]

选项：

[code]AllowOverride None[/code]

修改为（允许.htaccess文件Override设置）：

[code]AllowOverride All[/code]

2、配置PHP

  • 一般情况下修改Httpd配置文件：

[bash]vi /etc/php.ini[/bash]

选项disable_functions可阻止PHP运行高危函数：

[code]disable_functions = passthru,exec,system,chroot,chgrp,chown,shell_exec,proc_open,proc_get_status,ini_alter,ini_alter,ini_restore,dl,openlog,syslog,readlink,symlink,popepassthru,stream_socket_server,escapeshellcmd,dll,popen,disk_free_space,checkdnsrr,checkdnsrr,getservbyname,getservbyport,disk_total_space,posix_ctermid,posix_get_last_error,posix_getcwd, posix_getegid,posix_geteuid,posix_getgid, posix_getgrgid,posix_getgrnam,posix_getgroups,posix_getlogin,posix_getpgid,posix_getpgrp,posix_getpid, posix_getppid,posix_getpwnam,posix_getpwuid, posix_getrlimit, posix_getsid,posix_getuid,posix_isatty, posix_kill,posix_mkfifo,posix_setegid,posix_seteuid,posix_setgid, posix_setpgid,posix_setsid,posix_setuid,posix_strerror,posix_times,posix_ttyname,posix_uname[/code]

选项open_basedir可限制PHP文件夹权限（此处限制到默认html文件夹以及tmp临时目录）：

[code]open_basedir = /var/www/html/:/tmp/[/code]

至此RHEL7服务器配置基本完成，关于SELinux，本人并不建议关闭，可根据实际应用权限修改相应的设置以达到使安全性、可用性平衡的目的。
