CentOS7 修改DNS，CentOS-Base.repo原生源

Posted on 2016-08-04 | In System |

前些日子买了企鹅的香港云服务，作为此博客的架设和SS之用，但是蛋疼的TX为了GFW设置了内地的DNS地址，导致Google等域名无法解析。

于是修改先修改DNS：

[bash]nmcli connection show System\ eth0[/bash]

[bash]nmcli connection modify System\ eth0 ipv4.dns "8.8.4.4 8.8.8.8"[/bash]

[bash]nmcli connection up System\ eth0[/bash]

修改完，yum update一下，咦？


大意就是无法解析mirrors.tencentyun.com，经查证是个内网源。 8-O

那么，就要修改yum源为CentOS自带的，163也可以但是由于在HK并且个人特殊的JP，所以用原生的。

这里提供个原生的CentOS-Base.repo吧，每次找也麻烦。

[code]

CentOS-Base.repo

#

The mirror system uses the connecting IP address of the client and the

update status of each mirror to pick mirrors that are updated to and

geographically close to the client. You should use this for CentOS updates

unless you are manually picking other mirrors.

#

If the mirrorlist= does not work for you, as a fall back you can try the

remarked out baseurl= line instead.

#
#

[base]
name=CentOS-$releasever - Base
mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=os&infra=$infra

#baseurl=http://mirror.centos.org/centos/$releasever/os/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7

#released updates
[updates]
name=CentOS-$releasever - Updates
mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=updates&infra=$infra

#baseurl=http://mirror.centos.org/centos/$releasever/updates/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7

#additional packages that may be useful
[extras]
name=CentOS-$releasever - Extras
mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=extras&infra=$infra

#baseurl=http://mirror.centos.org/centos/$releasever/extras/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7

#additional packages that extend functionality of existing packages
[centosplus]
name=CentOS-$releasever - Plus
mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=centosplus&infra=$infra

#baseurl=http://mirror.centos.org/centos/$releasever/centosplus/$basearch/
gpgcheck=1
enabled=0
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-7
[/code]

编辑/etc/yum.repos.d/CentOS-Base.repo，写入，完工。

记一次海淘 x99 主板 到手点不亮 以及解决

Posted on 2016-07-28 | In Hardware |

整个六月，闲来无事逛逛张大妈，突然发现了这么一记剁手贴：Prime会员专享： GIGABYTE 技嘉 GA-X99P-SLI 主板 $182.74（约1300）

想来亚马逊间隔了几年又给了一次 Prime 的体验机会，于是就剁剁剁，但是就此产生的问题真是不少。首先是价格，虽然 x99 便宜了，但是其对应的 CPU 一直居高不下，毕竟5、6代还算是比较新出的，还有搭配的 DDR4 内存高频的价格简直不敢恭维。还有就是我根本没有考虑到 x 系列是不带核显的，这个在黑苹果时候再说。
其次是购买时候，此主板的 BIOS 还未更新支持 6 代的 CPU，由此到手后真是折腾了一番，以下主要记录此次折腾。

首先，收到主板和 U 之后，由于内存还在转运坑，于是京东淘了条先试机。收到内存，兴奋的装完机后，点亮。。。怎么重启了。。。

还没见到自检画面即 reboot，这个问题在我看来有几种可能：

1、CPU 插座接触不良（重新上）
2、电源问题（短接，负载）
3、主板有缺陷，运输途中损坏（这个比较悲催）
一个个试过去，详细过程就不记了，简单的操作网上都有。结果问题依旧存在，由于不插内存时触发重启间隔缩短，由此确定很有可能问题出在 CPU 上，由于 CPU 坏的案例很少（我是没坏过），那就暂时把问题缩小到 CPU->MotherBoard 之间。
逛了下技嘉的官网，由于这款未在国内上市，在其美帝官网上找到了支持，看了下，嗯。。6月中旬更新的 BIOS 才支持 6 系 CPU，这下好办了，问题就暂时转移到 BIOS 上（因为这种板子缺少 CPU微码 开不起的可能性很大）。
继续看 其兄弟 GA-X99-UD3P 支持 QFlash Plus，也就是可以通过优盘在无 CPU 的情况下升级 BIOS，在看手上这块的，What？只能QFlash，没有 Plus 支持也就是没有刷 BIOS 的 USB 口！！

这下比较棘手了，问题解决方案只有两种

1、带去本地电脑城，或者借一颗 5 系的 CPU 先点亮主板，升级 BIOS 后再换回
2、想到之前刷 Thinkpad x230 使用的编程器，也许可以用来直接刷呢？

由于方案1的实现需要出门，家里蹲不愿意出门，于是先执行方案2，活马当死马

方案2的进行需要以下步骤：

1、确认 主板 BIOS 芯片型号：MX25L12873F
2、确认 编程器支持的 芯片型号：未找到对应
3、确认封装，Google BIOS 芯片，找到对应定义图：SOP8 封装

以上三点都确认完了，没有找到对应支持也是心中有点发虚，不过都进行到这一步了，硬着上吧

1、连接 编程器与主板 BIOS，先不接供电脚。这里直接刷的 主BIOS
2、选一个针脚定义相同的 大概可以替换的芯片型号，这里我选的：MX25L12845E
3、开刷，提示接触不良，那么就给主板通电（就是一直重启的状态），目的是让其 BIOS 芯片有供电
4、再次检测，接触不良的针脚减少了，确认触电接触良好后，也只能先不管，让他读两遍芯片，校验正确后，说明连接是稳定的
5、观察读出固件的头部，对比官方的 BIOS，相同，由此推断是可以刷入的（这里注意了，主板商提供的 升级BIOS，与编程器刷的 BIOS 是不同的，一般来说 升级BIOS 是不会替换原有 MAC SN 之类信息的，用编程器强制刷入，会造成清空，MAC 变为 88:88:88:88:87:88 之类）
6、备份好原厂 BIOS，刷入 升级BIOS，然后。。

7、Boot！出乎意料的点亮了！
8、由于 强刷了 BIOS，步骤5 中提到的问题出现，接着解决这个问题
8.1、先用技嘉的 @BIOS 尝试刷回 步骤6 中备份的 BIOS，提示无法降回老版本
8.2、尝试使用 Flashrom 强刷，由于是强刷非官方的升级，预测原来的 MAC 等信息应该会一同刷回
8.3、强刷完成，千万不要重启，否则又回到原点了，紧接着再次使用 步骤8.1 中的 @BIOS，升级 F22 BIOS，此时由于是官方升级 应该会保留信息
8.4、完成重启，进 BIOS 查看 MAC 发现已恢复出厂！

至此，CPU 兼容问题解决，谨以此记录。

搭建私人 Ngrok 转发通道

Posted on 2016-07-02 | In FxxkGFW |

不知什么时候，锅内电信把个人pppoe上网的80端口都给封了，更有甚者加入了大内网。于是，苦逼的猿们调微信时候就更崩溃了。。
好在网上有不少内网穿透工具，说白了就是从内网主动发送消息，穿透网关，那么必须就得有一台位于公网的Server来转发消息。具体原理可百度。

众多网站提供的转发服务中，有花生棒之类的限流量，也有不限制但是带宽不足的，毕竟这么多人挤在一起想不慢都难。于是乎，本子自己动手丰衣足食的想法，自己搭建一个吧。

ngrok是这类工具中比较出名的一个了，可惜的是官方已经放弃了开源的1.x版本，2.x版本是官方提供做商业用。关于2.x版本的使用，可以参照这篇文章：http://www.jianshu.com/p/a0a34f770837（PS：简书不错，不像很多网站到处粘贴格式都看不舒服）。

1、下载ngrok 1.x源码：

[bash]git clone https://github.com/inconshreveable/ngrok.git[/bash]

2、生成证书，比如要使用xxx.tunnel.meanevo.com，则此处域名填写tunnel.meanevo.com

[bash]openssl genrsa -out rootCA.key 2048
openssl req -x509 -new -nodes -key rootCA.key -subj "/CN=填写域名" -days 5000 -out rootCA.pem
openssl genrsa -out server.key 2048
openssl req -new -key server.key -subj "/CN=填写域名" -out server.csr
openssl x509 -req -in server.csr -CA rootCA.pem -CAkey rootCA.key -CAcreateserial -out server.crt -days 3650[/bash]

3、替换掉原来的cert

[bash]cp rootCA.pem assets/client/tls/ngrokroot.crt
cp server.crt assets/server/tls/snakeoil.crt
cp server.key assets/server/tls/snakeoil.key[/bash]

4、配置Go语言环境，编译参数：
4.1、安装Go：

[bash]yum install go[/bash]


4.2、这里可以两种方式：
4.2.1、软连接
ln -s /usr/local/go/bin/* /usr/bin/
4.2.2、
export GOROOT=/usr/local/go
export PATH=$PATH:$GOROOT/bin
4.3、
export GOPATH=/usr/local/ngrok/

4.4、设置编译参数：
服务端32位：

[bash]GOOS=linux GOARCH=386[/bash]

服务端64位：

[bash]GOOS=linux GOARCH=amd64[/bash]

MacOS：

[bash]GOOS=darwin GOARCH=amd64[/bash]

Windows：

[bash]GOOS=windows GOARCH=amd64[/bash]

5、编译server和client，注意client得用自己编译出来的，有些现成的连不上，可能是证书原因：

[bash]make release-client
make release-server
make release-all[/bash]

不同平台：

[bash]GOOS=darwin GOARCH=amd64 make release-client
GOOS=linux GOARCH=amd64 make release-server
[/bash]

6、运行：
6.1、服务端：

[bash]./bin/ngrokd -domain="tunnel.meanevo.com" -httpAddr=":8080" -httpsAddr=":8081" -tunnelAddr=":4443"[/bash]

第一个是服务器端的http映射地址，第二个是https，第三个是控制端口。若需直接访问请配通firewall。
6.2、客户端：
6.2.1、创建一个ngrok.cfg，内容：

[code]server_addr: "tunnel.meanevo.com:4443"
trust_host_root_certs: false[/code]

6.2.2、启动：

[bash]ngrok -subdomain 相对二级域名地址 -config=ngrok.cfg 80[/bash]

这里的相对二级域名就是xxx.tunnel.meanevo.com的xxx

7、其他及优化：
7.1、域名DNS：将*.tunnel.meaneovo.com改A记录至服务器IP
7.2、个人建议在访问前再加一层nginx反向代理，使用特定域名访问外网80转发到ngrok
7.3、ngrok说白了是个被抛弃的项目，官方也说了，存在很大的性能以及内存泄漏问题，并且不会被（官方）修复，可以试试寻找github上修改好的的开源项目
7.4、可用于路由ngrok连接（需修改源码，附）
7.5、分享一个启动脚本，可添加此脚本至rc.local实现开机启动

用于Koolshare固件连接，修改源码（https://github.com/koolshare/ngrok-1.7/blob/master/src/ngrok/server/tls.go#L41）：
1、修改src/ngrok/server/tls.go，第41行添加:

[bash]vi src/ngrok/server/tls.go[/bash]


2、重新编译。
3、启动：

启动脚本（注意修改path、证书位置）：

[bash]domain="tunnel.meanevo.com"
http_port=8080
https_port=8081
remote_port=4443
path=/usr/share/ngrok
cert_path=$path/assets/server/tls

NPID=ps -ef|grep ngrokd|grep -v grep|awk '{print $2}'
if [ -z "$NPID" ]; then
echo "Not running, starting…"
else
echo "Already running, restarting…"
kill $NPID
fi

nohup $path/bin/ngrokd -domain="$domain" -httpAddr=":$http_port" -httpsAddr=":$https_port" -tlsCrt=$cert_path/snakeoil.crt -tlsKey=$cert_path/snakeoil.key -tunnelAddr=":$remote_port" > /var/log/ngrok.log 2>&1 &
sleep 1
head /var/log/ngrok.log
[/bash]

关于CentOS爬墙那点事，CentOS7 VPN－ShadowSocks

Posted on 2015-08-21 | In FxxkGFW |

接上文。接着来实现第二种方式：ShadowSocks。

今天听说软件作者被请去喝茶了，不过这种思路估计不容易被封锁吧。

项目首页（https://github.com/shadowsocks/shadowsocks）

如今SS安装使用已经很简单，几句命令而已，轻量级的优点吸引了不少人。

1、安装：

[bash]yum install python-setuptools && easy_install pip[/bash]

[bash]pip install shadowsocks[/bash]

2、启动：

[bash]sudo ssserver -p 443（端口） -k test!ss（密码） -m aes-256-cfb（加密方式） –user nobody（运行用户）-d start[/bash]

3、添加防火墙规则：

[bash]vi /etc/sysconfig/iptables[/bash]

[bash]-A INPUT -p tcp -m state –state NEW -m tcp –dport 443 -j ACCEPT[/bash]

4、测试，添加至启动项：

[bash]echo "ssserver -p 443（端口） -k test!ss（密码） -m aes-256-cfb（加密方式） –user nobody（运行用户）-d start" >> /etc/rc.local[/bash]

[bash]chmod +x /etc/rc.d/rc.local[/bash]

5、各个平台也几乎都有了SS客户端，输入服务器地址，密码， ENJOY!

其他几点以及一些实用的优化：

1、关于SS的管理平台

  • 用的比较多的就是ss-panel以及whmcs，个人使用没必要安装。

2、放一个查看当前连接的脚本，因为是按需连接，所以只能看到当前连接在用的用户：

[bash]netstat -tun|grep 端口号|sort|awk ‘{print $5}’|cut -d : -f 1|uniq[/bash]

放到PHP里就是:

[php]<?php exec("netstat -tun|grep 端口号|sort|awk ‘{print $5}’|cut -d : -f 1|uniq", $ips); var_dump($ips); ?>[/php]

3、对于linux3.0以上内核，可以开启fast-open，启动命令中添加:

[bash]sudo ssserver -p 443（端口） -k test!ss（密码） -m aes-256-cfb（加密方式）–fast-open –user nobody（运行用户）-d start[/bash]

当然客户端也要开启，添加到客户端配置json文件：

[bash]"fast_open": true,[/bash]

4、关于TCP优化，开启Hybla算法，聊胜于无，按照网上配置吧，实测Vultr还是不优化快一点，根据使用环境按需使用：

[bash]vi /etc/sysctl.conf[/bash]

[bash]net.ipv4.tcp_syncookies = 1

net.ipv4.tcp_tw_reuse = 1

net.ipv4.tcp_tw_recycle = 0

net.ipv4.tcp_fin_timeout = 30

net.ipv4.tcp_keepalive_time = 1200

net.ipv4.tcp_max_syn_backlog = 8192

net.ipv4.tcp_max_tw_buckets = 5000

net.ipv4.tcp_fastopen = 3

net.ipv4.tcp_rmem = 4096 87380 67108864

net.ipv4.tcp_wmem = 4096 65536 67108864

net.ipv4.tcp_mtu_probing = 1

net.ipv4.tcp_congestion_control = hybla

[/bash]

启用Hybla模块:

[bash]/sbin/modprobe tcp_hybla[/bash]

应用更改:

[bash]sysctl -p[/bash]

关于CentOS爬墙那点事，CentOS7 VPN－SoftEther VPN Server

Posted on 2015-08-21 | In FxxkGFW |

虽说一直买着主机搭梯子用，但是由于最近一直用的AWS到期，ConoHa限速512Kb的事情，梯子终于断了。

重新找了家Vultr搭，速度还不错，分享下。

首先，要使用上真正的Internet，现在比较流行的就是两种方式了

  • 自己搭建传统VPN，即通过PPTP、L2TP、SSTP、OpenVPN甚至使用IPSec来实现流量转发；
  • 使用Shadowsocks等Socks5代理。
    先说传统VPN吧，这种方式用得最多，设备支持也比较好，撇开PPTP这种现在看起来加密不太好的，L2TP、SSTP还是比较坚挺，

为了方便，就不再细说搭建PPTP服务器了，网上资料也很多，搭建也不难。

本文第一部分就先介绍下SoftEther这个东西（以下简称SE）。在没有这个软件的时候，L2TP搭建还是比较麻烦的事情，更不用提OVPN了。

官方网址：http://www.softether.org（可能需要爬墙）

SE是由日本筑波大学（University of Tsukuba）开发的免费软件，集成了L2TP、L2TP over IPSec、SSTP、OVPN服务端，并且管理工具的存在（SoftEther VPN Server Manager），使得设置变的非常简单，添加帐号、限速、NAT设置都有图形化界面。

下面说下安装。

1、先进入官方的下载页面（http://www.softether-download.com/en.aspx，可能需要爬墙），软件选择VPN Server，目标平台，CPU类型，然后复制最新版本的下载连接。如图所示：

2、切换到主机，cd 到/usr目录下新建一个文件夹，就叫SoftEther吧：

[bash]mkdir -p /usr/SoftEther &amp;amp;&amp;amp; cd /usr/SoftEther[/bash]

3、直接从官方下载tar.gz包并解压：

[bash]wget&amp;nbsp;http://www.softether-download.com/files/softether/v4.18-9570-rtm-2015.07.26-tree/Linux/SoftEther_VPN_Server/64bit_-_Intel_x64_or_AMD64/softether-vpnserver-v4.18-9570-rtm-2015.07.26-linux-x64-64bit.tar.gz[/bash]

[bash]tar -zxvf&amp;nbsp;softether-vpnserver-v4.18-9570-rtm-2015.07.26-linux-x64-64bit.tar.gz[/bash]

4、ll一下，可以看到自动创建了一个vpnserver目录，并且已经具有执行权限，进入这个文件夹，编译（先安装gcc）：

[bash]yum install gcc[/bash]

[bash]make[/bash]

失败请确认Root权限！

5、在一路按1同意了几条许可后，安装成功，提示中写明了启动命令（./vpnserver start），不过先不急着启动：

6、启动之前，先得添加iptables规则吧 XD，添加TCP端口443，992，1194，5555和UDP 1701端口（L2TP），记得如果路由上有VPN穿透把他们打开，GRE、ESP等等：

[bash]vi /etc/sysconfig/iptables[/bash]

添加：

[code]
-A INPUT -p tcp -m state –state NEW -m tcp –dport 443 -j ACCEPT
-A INPUT -p udp -m state –state NEW -m udp –dport 500 -j ACCEPT
-A INPUT -p tcp -m state –state NEW -m tcp –dport 992 -j ACCEPT
-A INPUT -p tcp -m state –state NEW -m tcp –dport 1194 -j ACCEPT
-A INPUT -p udp -m state –state NEW -m udp –dport 4500 -j ACCEPT
-A INPUT -p tcp -m state –state NEW -m tcp –dport 5555 -j ACCEPT
[/code]

7、OK，在防火墙重新启动之后，我们开启vpnserver服务：

[bash]systemctl restart iptables.service[/bash]

[bash]/usr/SoftEther/vpnserver/vpnserver start[/bash]

8、下载、使用管理工具（现在只有Win端，安装选择带有管理工具的）（2016.08更新，MacOS也有了）：

点击建立新设置 New Setting：


第一次连接会让你设置密码：

9、初始设置，选择最后一项跳过后点确定设置L2TP：

10、添加用户，设置SecureNAT或者直接桥接网卡：

  1. 一般使用SecureNAT，也就是接入后，分配设置地址池地址，然后NAT出到公网；
  2. 桥接的话，用途比较特殊，一般用于内网机器，并且需要局域网开启混杂模式，每次接入都由桥接网卡上游DHCP分配地址。

打开NAT，默认的是192.168.30.0/24网段，可以自行更改：

11、测试连接。

12、测试无误后，设置开机启动：

[bash]echo "/usr/SoftEther/vpnserver/vpnserver start" >> /etc/rc.d/rc.local[/bash]

[bash]chmod +x /etc/rc.d/rc.local[/bash]

至此SoftEther搭建完成。
